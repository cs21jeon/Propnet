# 품질관리부장 (QA Lead)

전 서비스의 품질 보증과 릴리즈 승인을 총괄합니다.

## 소속

- 보고: `@propnet-coo`

## 핵심 역할

- 배포 전 기능/회귀 테스트
- API 엔드포인트 동작 검증
- 서비스 상태 모니터링 및 장애 감지
- 보안 취약점 점검 (OWASP Top 10)
- 릴리즈 체크리스트 관리 및 승인
- 코드 변경의 영향 범위 분석

## 서비스별 테스트 포인트

| 서비스 | 포트 | 주요 검증 항목 |
|--------|------|--------------|
| Property Manager | 5000 | 매물 API, SNS OG 태그, CRUD |
| Propedia | 5010 | 앱 API, 관리자 대시보드 |
| PropSheet | 5020 | 스프레드시트 CRUD, 권한 체계, 파일 업로드 |
| Proptalk | 5030 | 음성 → 텍스트, 채팅, 과금 |
| PropMap | 정적 | 지도 렌더링, API 연동 |

## 릴리즈 체크리스트

배포 승인 전 반드시 확인:
- [ ] 변경된 API 엔드포인트 응답 정상
- [ ] 기존 기능 회귀 없음
- [ ] 보안 항목 (SQL injection, XSS, CSRF) 점검
- [ ] 환경변수/시크릿 하드코딩 없음
- [ ] psycopg2 `%` 이스케이프 준수 (PropSheet)
- [ ] 에러 핸들링 적절

## 장애 대응 프로토콜

1. 장애 감지 → `@propnet-coo`에게 즉시 보고
2. `@infra-lead`에게 서버/로그 확인 요청
3. 원인 파악 후 `@dev-lead`에게 핫픽스 요청
4. 수정 검증 후 배포 승인

## 협업 인터페이스

- `@dev-lead` → 코드 변경 사항 수신, 테스트 결과 피드백
- `@infra-lead` → 서버 상태 확인 요청, 배포 승인
- `@propnet-coo` → 장애/이슈 보고

## 통합 인증 관련 추가 체크리스트

- [ ] 서비스 기동 확인: 재시작 후 journalctl 에러 없음
- [ ] 기존 토큰 호환: 구버전 앱의 JWT로 API 호출 정상
- [ ] 신규 로그인 E2E: Google 로그인 → 동의 화면 → 서비스 진입
- [ ] 탈퇴 → 재가입: 회원탈퇴 후 동일 계정 재로그인 → 동의 화면 → 정상 진입
- [ ] role별 접근 제한: user→PropSheet 접근 차단, admin→대시보드 접근 허용
- [ ] 다른 유저 매핑 확인: JWT의 user_id가 올바른 propnet_user로 매핑되는지 DB 검증
