# UX 디자인 — PropMap 동별 클러스터링 지도

> 작성일: 2026-04-16

> 연관: `docs/prd-propmap-dong-clustering.md`, `docs/tech-design-dong-clustering.md`

---

## 1. UX 핵심 원칙

1. **정보 밀도는 줌에 비례** — 멀리서는 요약, 가까이서는 상세
2. **파크리오 같은 대단지는 "단지 1개" → "16개 동"의 자연스러운 전개**
3. **마커 색상은 기존 9종 팔레트 유지** (일관성 > 신규성)
4. **매물 리스트는 기존 PropMap 카드 재사용** (학습비용 제로)
5. **줌 전환은 200ms 이내 애니메이션으로 연속감 제공**

---

## 2. 줌 레벨 전환 규칙

| 줌 레벨 | 마커 모드 | 라벨 | 동작 |
|---|---|---|---|
| 5–13 | 시·구 단위 집계 (기존) | 구명 | 기존 동작 유지 |
| 14 | 동 단위 집계 (기존) | 법정동명 | 기존 동작 유지 |
| **15** | **단지(complex) 마커** — 새 동작 | 단지명 + "N건" | 클릭 시 단지 전체 리스트 |
| **16–17** | **동(dong) 마커** — 새 동작 | 동명 + "N건" | 클릭 시 해당 동 리스트 |
| 18+ | **동 마커 + 상세 라벨** | 건물명 + 동명 + "N건" | 좌동 |

> 줌 15 경계에서 complex → dong 전환. 이 임계값은 지도 경험상 "단지 전체가 화면에 보이는 순간"과 일치.

## 3. 마커 디자인

### 3-1. 단지 마커 (complex)

```
┌─────────┐
│  [16]   │  ← 단지 내 매물 개수 (큰 숫자)
│  파크리오 │  ← 단지명 (작은 텍스트, optional)
└─────────┘
```

- 크기: 48×48px (모바일) / 56×56px (데스크톱)
- 모양: 둥근 사각형 (radius 12px) + 하단 꼬리
- 색상: 기존 9종 팔레트에서 매물 수 구간별 할당
  - 1–5건: 팔레트 1
  - 6–15건: 팔레트 4
  - 16+건: 팔레트 7 (가장 진함)

### 3-2. 동 마커 (dong)

```
┌──────┐
│ [3]  │  ← 해당 동 매물 개수
│ 103동 │  ← 동명
└──────┘
```

- 크기: 36×36px (모바일) / 40×40px (데스크톱)
- 모양: 원형 + 동명 라벨(아래)
- 색상: 단지별로 동일 색상 계열 유지 (같은 단지는 같은 색)

### 3-3. 매물 없는 동

- 파크리오 16개 동 중 매물이 있는 동만 마커 표시
- 줌 18+에서는 매물 없는 동도 옅은 회색 윤곽선으로 표시 (선택 — B안)

### 3-4. 기존 9종 팔레트 유지

```
palette[0]  #E63946  Rose
palette[1]  #F4A261  Amber
palette[2]  #E9C46A  Sand
palette[3]  #2A9D8F  Teal
palette[4]  #457B9D  Steel
palette[5]  #264653  Deep
palette[6]  #8E6CEF  Violet
palette[7]  #118AB2  Cobalt
palette[8]  #06D6A0  Mint
```

(기존 PropMap 팔레트 — 변경 없음)

## 4. 줌 전환 애니메이션

### 4-1. complex → dong (줌 인)

```
0ms:   단지 마커 (스케일 1.0, 투명도 1.0)
100ms: 단지 마커 (스케일 0.7, 투명도 0.3) + 동 마커 페이드인 (투명도 0 → 0.7)
200ms: 단지 마커 제거 + 동 마커 (스케일 1.0, 투명도 1.0)
```

### 4-2. dong → complex (줌 아웃)

역순 200ms 애니메이션. 동 마커들이 단지 중심으로 흡수되는 스프링 이징.

### 4-3. 성능 고려

- `requestAnimationFrame` 기반
- 마커 100개 이상 시 CSS `will-change: transform` 적용
- 지도 SDK의 CanvasLayer 사용 (DOM 마커 X)

## 5. 클릭 팝업 레이아웃

### 5-1. 단지 팝업 (complex)

```
┌───────────────────────────────────┐
│ 잠실 파크리오            [닫기 ×]  │
│ 송파구 신천동 7                   │
│ 총 16건 · 16개 동                │
├───────────────────────────────────┤
│ [매물카드 1]  [매물카드 2]         │
│ [매물카드 3]  [매물카드 4]         │
│ …                                 │
├───────────────────────────────────┤
│   [동별로 보기 ↑ 줌인]             │
└───────────────────────────────────┘
```

- 폭: 360px (모바일 full-width)
- 매물카드는 기존 PropMap 카드 그대로 재사용
- 하단 CTA: "동별로 보기" → 지도를 줌 16으로 이동

### 5-2. 동 팝업 (dong)

```
┌───────────────────────────────────┐
│ 잠실 파크리오 103동       [닫기 ×] │
│ 송파구 신천동 7                   │
│ 총 3건                            │
├───────────────────────────────────┤
│ [매물카드 1]                      │
│ [매물카드 2]                      │
│ [매물카드 3]                      │
├───────────────────────────────────┤
│   [단지 전체 보기 ↓ 줌아웃]        │
└───────────────────────────────────┘
```

- 동 이동 UI: 좌우 화살표 버튼(103동 ↔ 102동)으로 같은 단지 내 다른 동 순환 탐색
- 하단 CTA: "단지 전체 보기" → 줌 15로 이동

## 6. 상태별 UI

| 상태 | 표시 |
|---|---|
| 로딩 | 지도 위 우상단 스피너 "매물 불러오는 중" |
| 데이터 없음 | bbox 내 매물 0건 → 중앙 "이 영역에 매물이 없어요" 토스트 |
| API 실패 | "일시적으로 지도를 불러오지 못했어요. 잠시 후 다시 시도해주세요." + 재시도 버튼 |
| 폴백 좌표 | 마커에 작은 경고 아이콘 (옅은 주황) — "건물 정보가 일부 누락되었습니다" 툴팁 |

## 7. 반응형

| 브레이크포인트 | 마커 크기 | 팝업 | 비고 |
|---|---|---|---|
| < 768px (모바일) | 작게 (36/48) | full-screen bottom-sheet | 매물카드 세로 스크롤 |
| 768–1024px (태블릿) | 중간 | 사이드 패널 (우측 400px) | 그리드 2열 |
| > 1024px (데스크톱) | 표준 (40/56) | 사이드 패널 (우측 440px) | 그리드 2–3열 |

---

## 8. 접근성

- 마커 min contrast ratio 4.5:1 (팔레트 검증 필요)
- 키보드 네비게이션: Tab으로 마커 순환, Enter로 팝업 열기
- 스크린리더: `aria-label="파크리오 16개 동 16건 매물"`
- 색각이상 대응: 색상 외 숫자 배지로도 구분 가능
