## 1. 배경 & 문제 정의

### 1-1. 현재 문제
Week 3 동 클러스터링으로 건물 단위 마커는 구현되었으나, **대단지 1개가 여러 지번(필지)에 걸쳐 있어 "단지명"이 아닌 "건물명"으로 흩어져 표시됨**.

#### 실제 케이스 (서울)
| 단지 | 지번 수 | 현재 표시 | 기대 표시 |
|---|---|---|---|
| 파크리오 (신천동) | 지번 17, 17-4, 17-5, 20, 20-6 | 지번별 5개 마커 또는 건물동별 66개 마커 | **단지 1개 "파크리오" 통합 마커** |
| 잠실주공5단지 (잠실동 27) | 지번 1개 | 공시=주공아파트5단지 / 건대=빈값 / 도로=주공아파트 507동 — **건물명 혼란** | **단지 1개 "잠실주공5단지"로 통합, 3종 별칭 검색 지원** |
| 장미 1/2/3차 (신천동 7,11) | 장미2·3은 **동일 PNU(11번지) 공유** | PNU 기반 묶음 시 장미2+3 합쳐짐 (오류) | **단지고유번호 기준으로 분리**, PNU는 1:N 허용 |

### 1-2. 근본 원인
- 우리 DB가 지번(`pnu`) 중심으로 설계됨
- "단지"라는 1차원이 비어있음
- 한국부동산원이 **"단지고유번호"를 공식 발급**하고 있으나 활용하지 않음

### 1-3. 해결 방향
한국부동산원이 공공데이터로 공개한 **"공동주택 단지 식별정보_기본정보" CSV**를 기반으로 `complex_master` 테이블을 구축하고, 매물·지도·자동완성 전반에 단지 차원 추가.

## 4. 주요 User Story

### US-1. 매물 등록 시 단지 자동완성
> 공인중개사 A가 Propedia 앱에서 "파크리오 102동 1203호"를 등록하려 한다.

**Given** 단지명 입력 필드에 "파크리"까지 입력
**When** 자동완성 드롭다운 표시
**Then** "파크리오 (서울 송파구 신천동 17, 66동 6864세대)" 상단 노출
**And** 선택 시 단지고유번호/대표 PNU/동 목록 자동 세팅
**And** 동 드롭다운에 "102동" 등 실제 존재하는 동명 로드

**완료 기준**:
- 검색어 2자부터 동작
- 응답 속도 < 200ms
- 키보드(↑↓Enter) 조작 가능

### US-2. 지도에서 대단지 1 마커
> 일반 방문자 B가 goldenrabbit.biz 매물 지도를 보고 있다.

**Given** 지도 줌 레벨 = 중간(카카오 level 4~5)
**When** 파크리오 영역 조회
**Then** 파크리오 단지 외곽에 대표 마커 1개 표시 ("파크리오 · 매물 12건")
**And** 클릭 시 단지 매물 리스트 사이드패널 열림
**And** 줌 인 → 동별 마커 + 개별 매물 마커로 자연 전환

### US-3. 단지명 별칭 검색
> 고객 C가 "잠실주공"으로 검색

**Given** 단지명 검색창에 "잠실주공" 입력
**When** 검색 실행
**Then** 현재 "잠실엘스" (구 잠실주공1단지), "잠실리센츠" (구 잠실주공2단지), "잠실주공5단지" 모두 검색 결과 포함
**And** 각 결과의 "구 단지명: 잠실주공N단지" 라벨로 컨텍스트 제공

### US-4. 단지 상세 뷰
> 방문자 B가 파크리오 마커 클릭

**Given** 파크리오 단지 선택
**Then** 단지 정보 패널:
- 단지명 (기본: 공시가격 기준)
- 별칭 (건축물대장, 도로명주소, 과거 단지명)
- 세대수, 동수, 사용승인일
- 단지 내 매물 리스트 (등록된 경우)
- 단지 경계(폴리곤) 지도 하이라이트

## 5. 기능 요구사항

### F1. 데이터 적재
- **F1-1** CSV 다운로드 자동화 (연 1회)
- **F1-2** `complex_master` 테이블 UPSERT (단지고유번호 PK 중복 처리)
- **F1-3** `complex_aliases` 3종 단지명 분리 저장
- **F1-4** `complex_parcels` 대표 PNU 1건 저장 (세부 지번은 사후 보강)
- **F1-5** 건축물대장 캐시(`building_dong_geometry`)와 주소 매칭 → 세부 PNU 사후 보강
- **F1-6** 이력정보 CSV로 `complex_aliases` (type=past) 추가 적재
- **F1-7** 동정보 CSV로 `complex_dong` 테이블 적재

### F2. API
- **F2-1** `GET /api/complex/lookup?complex_pk=xxx` — 단지고유번호로 단지 정보
- **F2-2** `GET /api/complex/lookup?pnu=xxx` — PNU로 소속 단지 역조회
- **F2-3** `GET /api/complex/lookup?lat=&lon=&radius_m=` — 좌표 반경 내 단지 목록
- **F2-4** `GET /api/complex/search?q=파크리오&limit=10` — 단지명 자동완성 (3종 별칭 포함)
- **F2-5** `GET /api/complex/{complex_pk}/properties?agent_slug=` — 단지 내 매물 리스트
- **F2-6** 캐시 헤더: `Cache-Control: public, max-age=3600` (단지 마스터는 연 1회 갱신)

### F3. 프론트엔드
- **F3-1** PropMap `dong-cluster-renderer.js`에 complex 레이어 추가
  - zoom >= 5: 단지 클러스터 마커
  - zoom = 3~4: 단지 경계 + 동 마커
  - zoom <= 2: 개별 매물 마커
- **F3-2** Propedia 매물 등록 폼에 **단지명 자동완성** 추가
  - 선택 시 주소·대표 PNU·동 목록 자동 채움
- **F3-3** 동 드롭다운 **필수** (집합부동산 카테고리)
- **F3-4** 단지 상세 팝업 (F2-5 응답 기반)

### F4. 운영
- **F4-1** 연간 갱신 스크립트: `scripts/week5_complex_master/update_from_latest_csv.sh`
- **F4-2** 신규 단지는 매물 등록 시 자동 추가 (CSV 없는 신축 아파트 대응)
- **F4-3** 단지명 변경 알림 — 법정동 변경 알림 시스템(`project_bjdong_change_alert`)과 통합

## 6. 비기능 요구사항

### NF1. 성능
- `complex_master` 전수 적재 < 30분 (COPY 또는 배치 INSERT 2000건/트랜잭션)
- `/api/complex/search` P95 < 150ms (GIN trigram 인덱스)
- `/api/complex/lookup` P95 < 80ms (단지고유번호 PK)

### NF2. 가용성
- CSV 다운로드 실패 시 기존 데이터 유지 (atomic 재적재)
- `/api/complex/*` 500 에러 시 지도/매물 기능 degrade하지 않음 (fallback = 기존 동 클러스터링)

### NF3. 보안
- `complex_master`는 공개 데이터 → 인증 불필요
- 매물 정보 결합 시 agent 격리 원칙 유지 (다른 agent 매물 노출 금지)

### NF4. 데이터 품질
- 307,407건 중 95% 이상이 단지명 3종 중 최소 1개 보유
- 대표 PNU가 VWorld 캐시와 매칭되는 비율 80%+ (나머지는 카카오 Local 보강)

## 7. 범위 & 우선순위

### Must (Week 5 필수)
- M1: `complex_master`, `complex_aliases`, `complex_parcels` DDL + 적재
- M2: `/api/complex/lookup` + `/api/complex/search`
- M3: PropMap complex 레이어
- M4: Propedia 단지 자동완성
- M5: 파크리오/잠실주공 검증

### Should (여력 있으면)
- S1: `complex_dong` 테이블 + 동 드롭다운 연동
- S2: `complex_aliases.past` (이력 CSV)
- S3: 단지 경계 폴리곤 (VWorld로 사후 생성)

### Could (Week 6+)
- C1: 단지 단위 실거래가 대시보드
- C2: 단지 단위 매물 통계
- C3: 공동주택 외(오피스텔/주상복합) 확장

### Won't (이번 Week 5 제외)
- 단독주택·다가구 묶음
- 단지 리뷰·커뮤니티 기능
- 과거 단지명 전수 마이그레이션 (기존 매물 텍스트는 유지, 신규 등록만 신규 체계)

## 10. 성공/실패 판정 기준

### 성공 조건 (모두 충족 시 Week 5 완료 선언)
1. 파크리오 (신천동 17) 줌 레벨 4~5에서 **단일 단지 마커** 표시
2. 잠실주공5단지 검색 시 "주공/잠실주공/507동" 3종 모두 매칭

- P95 > 500ms 지속 시 엔드포인트 비활성화 + 동 클러스터링 fallback
- 적재 후 기존 지도 로딩 에러 발생 시 complex 레이어 feature flag OFF
- 오너 UX 수용 기준 미달 시 Phase 4 재설계
