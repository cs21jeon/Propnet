# PropNet 네이버 블로그 종합 운영 계획

## Context

PropNet(프롭넷)은 부동산 기술 서비스 브랜드로, 3개 서비스를 운영 중:
- **Propedia** (부동산백과) - Flutter 부동산 백과사전 앱 (Google Play 출시, v1.0.2+8)
- **PropTalk** (프롭톡) - 음성→텍스트 STT + AI 요약 협업 플랫폼 (Google Play 테스트 중, v1.0.0+3)
- **PropSheet** (프롭시트) - 웹 기반 Airtable 대안 부동산 데이터 관리 (Phase 3+, 운영 중)

네이버 블로그(blog.naver.com/propnet)를 회사 홈블로그로 운영하여 서비스 홍보, 부동산 지식 공유, 개발 스토리를 장기적으로 발행하려 함. 기존 blog-content 폴더에 2개 글(건축물대장, 공시지가)이 이미 작성되어 있음.

**블로그 상태**: 이미 개설 완료 (blog.naver.com/propnet)
**포스팅 도구**: Playwright MCP (미설치 상태, 이번 세션에서 설정)
**이번 세션 목표**: Playwright MCP 설정 → 첫 글(프롭넷 소개) 작성 → 네이버 블로그 발행 시도

## 3. 네이버 SEO 전략 (2026)

### C-Rank (크리에이터 랭크) 최적화
- 부동산/PropTech 주제에만 집중 (비관련 주제 금지)
- 최소 주 3회 포스팅 (3개월간), 이후 주 2회
- 게시글당 최소 1,500~3,000자 (한글 기준)

### D.I.A. (Deep Intent Analysis) 최적화
- 첫 2문단에 핵심 답변 배치
- 키워드 포함 서술형 제목 (낚시성 X)
- 긴 글은 상단 목차 포함

### 경험 기반 콘텐츠
- 실제 앱 스크린샷 활용 (19+ 이미지 보유)
- 정부24 vs Propedia 비교 등 "직접 해봤습니다" 포맷
- 개발 과정 실제 이야기

### 핵심 타깃 키워드
**Tier 1 (고검색량)**: 건축물대장 조회/보는법, 공시지가 조회, 토지대장 조회, 건폐율 용적률

**Tier 3 (롱테일)**: 건축물대장 앱 조회, 부동산 회의록 자동작성, 부동산 데이터 관리 엑셀 대안

## 6. 장기 성장 로드맵

### Phase 1: 기반 구축 (1~3개월)
- 36편 발행 (주 3회)
- 블로그 이웃 50명+ 추가 (부동산 블로거)
- Naver Search Advisor 등록
- 매일 이웃 블로그 5곳 이상 댓글 소통

### Phase 2: 성장 (4~6개월)
- 주 2회 고퀄리티 포스팅 (2,500자+)

- 네이버 VIEW 탭 노출 시작
- Naver 지식인 답변으로 블로그 유입

### Phase 3: 권위 (7~12개월)
- "부동산 앱", "건축물대장 조회" 등 핵심 키워드 VIEW 탭 TOP 10
- 블로그 구독자 500명+
- 월간 데이터 리포트 시리즈화
- 앱 다운로드 UTM 추적으로 전환율 측정

## 8. Playwright MCP 설정 가이드 (블로그 자동 포스팅)

### 개념 설명
Playwright MCP는 Microsoft가 만든 브라우저 자동화 도구(Playwright)를 Claude Code에서 직접 사용할 수 있게 해주는 MCP 서버. Claude가 실제 브라우저를 열어서 네이버 블로그에 로그인하고, 글을 작성하고, 이미지를 업로드하고, 발행까지 할 수 있음.

### Step 1: Playwright MCP 설치

```bash
# Claude Code 터미널에서 실행 (propedia 프로젝트 디렉토리에서)
claude mcp add playwright -- npx @playwright/mcp@latest
```

이 명령어는 `.mcp.json`에 자동으로 Playwright MCP 서버 설정을 추가함.

### Step 2: .mcp.json 설정 확인/수정

기존 `C:\Users\ant19\projects\propedia\.mcp.json`에 playwright 항목이 추가됨:

```json
{
  "mcpServers": {
    "goldenrabbit-server": {
      "command": "ssh",
      "args": [
        "-o", "LogLevel=ERROR",
        "-o", "StrictHostKeyChecking=no",
        "root@175.119.224.71",
        "npx -y @modelcontextprotocol/server-filesystem /home/webapp/goldenrabbit"
      ]
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

### Step 3: 브라우저 바이너리 설치

```bash
npx playwright install chromium
```

### Step 4: Claude Code 재시작 후 확인

```bash
# Claude Code 내에서
/mcp
```
→ `playwright` 서버가 connected 상태인지 확인

### 네이버 블로그 포스팅 워크플로우

Playwright MCP 설정 완료 후, Claude에게 다음과 같이 요청:

**첫 사용 (로그인):**
```
Playwright MCP로 네이버 블로그(blog.naver.com/propnet)에 접속해줘.
로그인이 필요하면 로그인 화면에서 멈춰줘 (내가 직접 로그인할게).
```
→ Claude가 브라우저를 열고 네이버 로그인 페이지로 이동
→ 사용자가 직접 로그인 (보안상 비밀번호는 사용자가 입력)
→ 로그인 완료 후 Claude에게 "로그인 완료" 알림

**글 작성 요청:**
```
Playwright MCP를 사용해서 네이버 블로그에 새 글을 작성해줘.
제목: [제목]
카테고리: [카테고리명]
내용은 blog-content/01-건축물대장-보는-법.md 파일을 네이버 블로그 포맷에 맞게 입력해줘.
이미지는 screenshots/ 폴더에서 관련 스크린샷을 업로드해줘.
```

**주의사항:**
- 네이버 로그인은 보안 정책상 자동화가 제한될 수 있음 → 수동 로그인 후 세션 유지
- 네이버 블로그 에디터는 SmartEditor로, DOM 조작이 복잡할 수 있음
- 첫 시도 시 Claude와 함께 단계별로 진행하며 에디터 구조 파악 필요
- 이미지 업로드는 파일 선택 다이얼로그가 필요하므로 별도 처리 필요할 수 있음

### 대안 접근법 (Playwright MCP가 네이버 에디터와 호환 문제 시)

네이버 SmartEditor가 iframe/복잡한 DOM 구조로 자동화 어려울 경우:
1. **blog-content/ 폴더에 완성본 작성** → 수동으로 복사-붙여넣기
2. **네이버 블로그 API** (Open API) 활용 검토
3. **Playwright MCP로 기본 흐름만 자동화** (접속 → 글쓰기 버튼 → 제목 입력까지)하고 본문은 수동
